How to Set Up an Agent Using Copilot

Published: January 2026

Use this guide to set up an automation agent. Follow the steps to configure, test, and troubleshoot the agent in your environment.

What you'll need

  • Permission to create and manage agents
  • Access to the systems the agent will use
  • Defined task or workflow to automate
  • Authentication details for connected services
  • Stable network connection

Set up an agent

  1. Open your agent tool and go to Agents.
  2. Select Create agent.
  3. Enter a clear name and a short description.
  4. Choose how the agent starts: on a schedule, by an event, or manually.
  5. Add the actions the agent performs.
  6. Configure access and credentials for any services the agent uses.
  7. Save the agent and turn it on.

Optional settings

  • Schedule: set frequency and start time.
  • Error handling: set retries and timeouts.
  • Notifications: send alerts on success or failure.
  • Data rules: validate inputs and apply transformations.
  • Logging: choose level and retention.

Verify the agent is working

  • Check the agent status shows Active.
  • Run a test trigger and confirm it completes.
  • Review logs for successful runs and no errors.
  • Confirm expected output appears in the destination.
  • Monitor alerts to ensure notifications are received.

Troubleshooting

Agent does not start

  • Verify the agent is turned on.
  • Confirm triggers are configured and meet conditions.
  • Check logs for start or trigger errors.

Agent output is incorrect

  • Review action steps and data rules.
  • Test with known input and compare expected results.
  • Update mappings or transformations if needed.

Authentication fails

  • Confirm credentials are valid and have access.
  • Re-enter tokens or keys and save.
  • Retry the agent and check logs for details.
