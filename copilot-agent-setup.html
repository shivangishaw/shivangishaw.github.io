How to Set Up an Agent Using Copilot

Published: January 2026

Use this guide to set up an automation agent. Follow the steps to configure, test, and troubleshoot the agent in your environment.

What you'll need

  • Permission to create and manage agents
  • Access to the systems the agent will use
  • Defined task or workflow to automate
  • Authentication details for connected services
  • Stable network connection

Set up an agent using Copilot

  1. Open Agents.
  2. Select Create new agent.
  3. Enter a name and description.
  4. Choose a start type: Scheduled, Event, or Manual.
  5. Add actions the agent will perform.
  6. Configure Credentials for connected services.
  7. Select Save, then switch the agent On.

Optional settings

  • Schedule: set frequency and start time.
  • Error handling: set retries and timeouts.
  • Notifications: send alerts on success or failure.
  • Data rules: validate inputs and apply transformations.
  • Logging: choose level and retention.

Verify the agent is working

  1. Confirm the agent status shows Active.
  2. Run a test trigger and check it completes.
  3. Review logs and ensure no errors are reported.
  4. Validate output appears in the expected destination.
  5. Verify alerts or notifications are received.

Troubleshooting

Agent does not start

  • Issue: Agent does not run when expected.
  • Cause: Trigger not configured or agent is off.
  • Fix: Turn the agent on; verify trigger conditions.

Output is incorrect

  • Issue: Results are missing or wrong.
  • Cause: Mapping or transformation rules are misconfigured.
  • Fix: Review actions and data rules; test with known input.

Authentication fails

  • Issue: Connection errors to a service.
  • Cause: Credentials are invalid or expired.
  • Fix: Update tokens or keys, save, and retry.
